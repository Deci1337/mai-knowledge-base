```markdown
## Ключевые факты и инсайты
- Необходимо провести ручную сверку отчетов для проверки корректности обработки данных.
- Проблема заключается в том, что данные, которые приходят, не совпадают с ожидаемыми результатами.
- Важно получить сырые данные для сверки с отчетами, чтобы убедиться в правильности расчетов.
- Проблема может заключаться в источниках данных, которые не являются корректными.

## Решения
- Роман Вассильев будет сверять данные, полученные из платформы, с сырыми данными, чтобы убедиться в правильности расчетов.
- Денис должен подготовить выгрузку сырых данных для анализа.
- Команда согласовала необходимость проверки данных, чтобы выявить, где именно происходит ошибка в обработке.

## Изменения в процессах
- Внедрение ручной сверки данных как нового процесса для проверки отчетности.
- Создание чек-листа для тестирования функционала платформы на основе ТЗ.
- Определение задач для команды, включая выгрузку данных и ручные расчеты.

## Прочие договорённости
- Запланировать встречу на воскресенье для обсуждения прогресса и результатов сверки.
- Денис должен заполнить таблицу с данными, которую затем проверит Роман.
- Роман должен составить список функционала платформы на основе ТЗ и предоставить его команде.
```
