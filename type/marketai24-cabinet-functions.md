# Функционал личного кабинета MarketAI24 — пошагово

ВАЖНО: всё ниже — точные названия кнопок, полей и разделов, зафиксированные вживую на тестовом аккаунте с активным демо-доступом. На аккаунте НЕ был подключён ни один реальный магазин, поэтому все таблицы с данными показывали «Нет данных» / «0 товаров» — сама СТРУКТУРА экрана (кнопки, фильтры, колонки) от этого не меняется, но КАК ИМЕННО выглядят реальные заполненные данные — не проверено. Разделы дополняют type/marketai24-app-navigation.md, не дублируя его.

## Как добавить магазин (кабинет) — точные шаги
1. В левом меню открыть «Мои организации».
2. Нажать кнопку «Добавить организацию» (доступна только если куплена подписка или активен демо-доступ; на аккаунте без тарифа кнопки нет вообще).
3. В открывшемся окне «Добавить организацию» заполнить поля:
   - «Название» — название кампании/кабинета (свободный текст).
   - «Маркетплейс» — выпадающий список: Wildberries / Ozon / Yandex Market.
   - «API ключ» — вставить ключ маркетплейса. Рядом ссылка «Как получить ключ?» — при клике разворачивается пошаговая инструкция (для Wildberries подтверждено дословно):
     1. Откройте WB Seller → Настройки → Доступ к API
     2. Нажмите «Создать новый токен»
     3. Выберите тип: Персональный токен или Сервисный токен
     4. Включите все разрешения (поставьте галочки на все пункты)
     5. Скопируйте токен и вставьте в поле выше
     Плюс кнопки «Открыть настройки API в WB Seller →» и «Открыть PDF-инструкцию».
   - Кнопка «Проверить» — проверяет валидность введённого ключа (сам процесс проверки, что показывает при успехе/ошибке — не проверено, тестовый ключ не вводился).
   - Чекбокс «Индивидуальные условия» — «Для этой организации применяются индивидуальные условия» (что именно это меняет — не проверено).
   - Ссылка «Налоги настраиваются в разделе Настройки → Налоги».
4. Кнопка «Создать» — завершает добавление. Что происходит дальше (сразу ли появляется кабинет в списке, сколько идёт первая синхронизация данных) — НЕ ПРОВЕРЕНО, реальный ключ не вводился намеренно (тестовый аккаунт).

## Как посмотреть статистику — главный дашборд «ОПиУ · Dashboard»
Путь: «Главное» в меню, либо «Управленка → ОПиУ · Dashboard».
Кнопки сверху: «Выбрать блоки» (кастомизация набора показанных метрик), «Скачать Excel» (экспорт), «Инвест.», «Фильтры», «AI ассистент» (открывает боковую панель с ИИ-подсказками по этой же странице).
Фильтры (общие для всех разделов «Управленки», см. ниже): Период (по умолчанию последние 7 дней), Кабинет, Бренд, Категория, Артикул поставщика (поиск), Связка товаров.
Показатели на дашборде: Заказы (руб. и кол-во), Продажи (руб. и кол-во), Маржинальная прибыль (+ кнопка «Задать план», если план не задан), Чистая прибыль, % Выкупа, Сумма возвратов, Штрафы, Комиссия, Эквайринг, К перечислению, Логистика, Внутренняя реклама, Платная приёмка, Средний чек, Кол-во возвратов, Кол-во отказов, Себестоимость, Хранение, Налоговая база, Налоги, Прочие удержания, Кэшбэк, Компенсация, Доля услуг маркетплейса. Ниже — «График метрик» с переключаемыми линиями (Продажи, Чистая прибыль, Заказы, Возвраты, Комиссия, Логистика, Хранение, Штрафы, Средний чек, Маржинальная прибыль).

## Общие фильтры во всех разделах «Управленки» — подтверждено
Один и тот же набор фильтров повторяется на каждой странице раздела: Период / Кабинет (выбор кампании) / Бренд / Категория / Артикул поставщика / Связка товаров. Знать этот список полезно, чтобы не путать разделы — они различаются только показателями и кнопками, набор фильтров у них одинаковый.

## ОПиУ · FIN отчёт — подтверждено
Путь: Управленка → ОПиУ · FIN отчёт. БЕЗ подключённого кабинета маркетплейса страница показывает только сообщение: «Нет кампаний — Создайте кампанию для просмотра финансовых отчетов». Содержимое отчёта при подключённом кабинете НЕ ПРОВЕРЕНО.

## История остатков — подтверждено
Путь: Управленка → История остатков. Кнопки: «Обновить», «Загрузить себестоимость», «Фильтры». Доп. переключатели: «По дням», «Сгорело», «Штуки», «Себестоимость», «Все склады», «Выгрузить в Excel». Таблица (без данных на тесте): колонки НАЗВАНИЕ, АРТИКУЛ, СРЕДНЕЕ.

## План-Факт — подтверждено
Путь: Управленка → План-Факт. Кнопки: «Обновить», «Загрузить план», «Выбрать блоки», «Фильтры».
Блок «Графики и аналитика»: график динамики (переключение «По дням» / «По неделям», линии Маржа / Заказы / Продажи) и «Нарастающий итог» (Маржа / Заказы / Продажи / Маржинальная прибыль / План маржинальность).
Блок метрик: по каждому показателю (Заказы, Маржинальная прибыль, Продажи) — три числа рядом: План / Факт / % Выполнение.
Таблица товаров: переключатель «Скрыть нулевые», единицы «Шт» / «₽», кнопка «Настроить таблицу».

## Юнит-экономика — подтверждено, самый детальный раздел
Путь: Управленка → Юнит-экономика. Кнопки: «Обновить», «Экспорт», «Загрузить себ.» (себестоимость), «Фильтры».
Режимы отображения (переключатели): «Сумма» / «На ед.» (на единицу товара), «Абс.» / «Доли» (абсолютные значения или проценты от выручки), «Факт» / «Гипотеза».
Таблица «Детализация по товарам (с гипотезой)» — 24 колонки: ТОВАР, ВЫРУЧКА, ВЫКУПЫ (шт.), СЕБЕСТ-ТЬ (H*), % СЕБЕСТ., КОМИССИЯ (H*), % КОМИССИИ, ЛОГИСТИКА (H*), % ЛОГИСТ., РЕКЛАМА (₽), ДРР %, ХРАНЕНИЕ, % ХРАН., ЭКВАЙРИНГ, % ЭКВАЙР., СТАВКА WB %, МАРЖА (H*), % МАРЖИ H*, Δ К ФАКТУ, НАЛОГ, % НАЛОГА, МАРЖА ФАКТ, % МАРЖИ ФАКТ, РЕНТАБЕЛЬНОСТЬ %.
Колонки с пометкой «(H*)» — редактируемые «гипотезы»: интерфейс прямым текстом поясняет «Редактируйте значения в колонках (H*) для пересчёта» — то есть можно вручную подставить другую себестоимость/комиссию/логистику/рекламу и сразу увидеть, как это изменит маржу — инструмент сценарного расчёта «что если», а не только фиксация факта.

## Что означает пометка «Подключить» у модулей Управленки — ТЕПЕРЬ ПОДТВЕРЖДЕНО
В прошлой версии документа это было не проверено. Сейчас проверено: клик по пункту с пометкой «Подключить» (например «ДДС», «РНП реклама», «Тепловая карта», «До поставки», «Регионы», «ABC-анализ», «База знаний») НЕ открывает сам раздел, а показывает модальное окно «Запрос на подключение»: «Вы запрашиваете доступ к: [название] — Нажмите кнопку ниже — мы получим вашу заявку и свяжемся с вами в ближайшее время» (кнопки «Отмена» / «Оставить заявку»). Это ОТДЕЛЬНАЯ ручная заявка на доступ, а НЕ то же самое, что «нет подключённого магазина» или «нужна подписка» — эти модули не открываются автоматически даже с активным демо-доступом и подключённым кабинетом, требуется just отдельно запросить доступ у команды. Список того, что было помечено «Подключить» на момент проверки: ДДС, РНП реклама, Тепловая карта, До поставки, Регионы, ABC-анализ, База знаний.

## Что по-прежнему НЕ проверено
- Как именно выглядят разделы с реальными данными (подключённый магазин с историей продаж)
- Что показывает «Проверить» при вводе реального/неверного API-ключа
- Что происходит сразу после нажатия «Создать» в форме добавления организации
- Раздел «Задачи» и «ИИ и автоматизация задач» — не открылись из-за модального окна, перекрывавшего клик; данных нет
- Содержимое «ОПиУ · FIN отчёт» при подключённом кабинете
Если вопрос менеджера касается этого — говорить прямо «нет данных», не додумывать по аналогии с другими разделами.
