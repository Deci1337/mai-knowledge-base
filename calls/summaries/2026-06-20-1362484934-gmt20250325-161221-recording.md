# Выжимка: Обсуждение структуры платформы MarketAI24 и ТЗ для разработки

**Тип:** team
**Дата:** 20.06.2026
**Сотрудник:** Arseniy

## Ключевые факты

- Созвон команды (Arseniy/Ром + Семён + другие) по обсуждению структуры платформы
- Платформа имеет два типа учёта: финансовый отчёт (точный) и управленческий (данные ВЛБС — не точный, но используется день-в-день)
- Основные разделы: дашборд (общие показатели по компании), опил по месяцам, опил по неделям, артикульный отчёт, управленческий отчёт (самый рабочий инструмент)
- Управленческий отчёт (Семён сделал) — читабельный, работает по связкам артикулов; менеджер видит только свои артикулы
- Обсуждается функционал: сравнение двух периодов (предложен спарклайн или отдельные таблички), конструктор связок, распределение доступов по ролям
- Семён уже реализовал управленческий отчёт; остальные страницы — по 2 дня на каждую
- Упоминается: Роман работает с АПИ Wildberries (данные по рейтингам — ещё тестирует), интеграция с сервисом бустов (живой)
- Обсуждается юнит-экономика: нужна отдельная вкладка, два способа ввода с/с (через файл и через форму); нужен конструктор с/с стоимости
- Упоминается сервис самовыкупа, который написал Влад — работает, одна бага с отзывами найдена

## Решения

- ТЗ делать в Miro: рисуешь структуру меню → от каждого пункта рисуешь экран → комментарии к каждой кнопке → ссылки для перехода
- По каждой странице описать: какие данные используются, откуда берутся
- Распределение доступа (роли) — описать текстом
- Сначала описать управленческий блок, потом расставить приоритеты совместно
- Добавить в таблицу тип действия «бартер» (помимо самовыкупа и внешней рекламы)
- ДДС также войдёт в меню платформы
- Интеграция с ПланФактом — отложить до после основного релиза
